Since you are using TypeScript, the good news is that the @google/adk package is
a TypeScript-first library and ships with its own type declarations. You do not
need to install a separate @types/... package.

The errors you are seeing are happening because your code is trying to import
from nested sub-directories (/agents and /tools). In the TypeScript version of
the Google Agent Development Kit, everything is exported directly from the root
module.

Here is exactly how to fix it for your TypeScript project:

1. Install the package

Ensure the actual package is installed in your project:

npm install @google/adk

(Use yarn add @google/adk or bun add @google/adk if you are using a different
package manager)

2. Update your TypeScript imports

Change your import paths to pull directly from '@google/adk'. Also, note that
the Google Search tool is exported as the constant GOOGLE_SEARCH in the
TypeScript SDK.

Change this:

// ❌ Incorrect
import { LlmAgent } from '@google/adk/agents'
import { googleSearch } from '@google/adk/tools'

To this:

// ✅ Correct
import { LlmAgent, GOOGLE_SEARCH } from '@google/adk';

// Example implementation:
const agent = new LlmAgent({
  name: 'research_assistant',
  model: 'gemini-3.1-flash',
  instruction: 'You are a helpful assistant. Answer user questions using Google Search when needed.',
  tools: [GOOGLE_SEARCH], 
});

3. Check tsconfig.json (If errors persist)

If you updated the imports and installed the package but your editor is still
showing a red squiggly line, make sure your TypeScript configuration is set up
for modern Node module resolution. Check your tsconfig.json for the following:

{
  "compilerOptions": {
    "moduleResolution": "node", /* or "Bundler" */
    "esModuleInterop": true
  }
}
